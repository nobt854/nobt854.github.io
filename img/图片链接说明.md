# 博客图片规则

当前仓库统一使用站内图片路径，不再把 `raw.githubusercontent.com` 作为默认方案。

## 标准写法

博客正文图片最终都应写成：

```markdown
![示意图](/img/blog/<文章-slug>/<文件名>)
```

例如：

```markdown
![Terminal 页签管理](/img/blog/claude-code-ide-workflow/image.png)
```

## 自动处理方式

仓库已提供自动同步脚本：

```bash
npm run sync:post-images
```

同时本地 Git 已接入 `.githooks/pre-commit`：

- 提交前会扫描本次暂存的 `_posts/*.md`
- 自动把图片复制到 `img/blog/<文章-slug>/`
- 自动把正文图片链接改写成 `/img/blog/<文章-slug>/<文件名>`
- 如果文章里引用了不存在的图片，提交会被拦住

## 草稿阶段可接受的写法

草稿里可以先写这些形式：

```markdown
![示意图](./demo.png)
![示意图](assets/demo.png)
![示意图](/assets/demo-post/demo.png)
```

提交前脚本会统一整理成站内标准路径。

## 历史说明

旧文章里部分图片使用过：

```text
https://raw.githubusercontent.com/nobt854/nobt854.github.io/master/...
```

这是历史兼容方案，不再作为新文章默认规则。
