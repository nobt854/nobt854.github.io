# Blog 写作规范

## 目标
- 整体博客美观
- 整体博客易读
- 根据主题内容生成的博客内容完整
- 整体不违和

## 视觉美化标准

### 1. 表格格式

✅ 使用 Markdown 原生表格（不用 ASCII 表格）

```markdown
| 维度 | 评分 | 说明 |
|------|------|------|
| 工具熟练度 | ★★★★★ | 713 条会话，深度定制 |
```

✅ 多列对比使用表格

```markdown
| 以前 | 现在 |
|------|------|
| "帮我写个贪吃蛇" | "这个 Java 服务的性能瓶颈在哪里" |
| 代码能跑就行 | 有 lint 有测试有文档 |
```

### 2. 章节标题

✅ 每个二级标题添加相关 emoji 图标

```markdown
## 🎮 第一阶段：玩具期
## 💡 第二阶段：觉醒时刻
## 🏗️ 第三阶段：体系化
## 🔍 核心洞察
## 📈 使用模式的演变
## ✂️ 下一阶段：精简
## 📝 给你的建议
## 🌅 尾声
```

### 3. 分隔线

✅ 使用 `---` 分隔主要章节

```markdown
---

## 章节标题
```

### 4. 引用块

✅ 重要观点使用引用块 + 加粗

```markdown
> **证据 > 速度**

> **技术驱动型 | 配置爱好者 | 深度使用者**

> **因为最好的系统，不是不能再加东西，而是不能再减东西。**
```

### 5. 列表格式

✅ 普通列表（3 个以内）

```markdown
> - "帮你写个贪吃蛇"
> - "这个按钮怎么居中？"
> - "解释一下这段代码"
```

✅ 带状态标记的列表

```markdown
- ✅ **完善的配置体系**
- ⚠️ **凭证管理需要优化**
- ⚠️ **配置可以精简**
```

### 6. 叙事风格
支持多种叙事风格的切换，默认采用技术原样输出风
- 技术原样输出风：根据输入的主题内容仅稍加修饰，大部分保持原样，增加全文博客的完整性
- 轻松幽默：在输入的主题内容基础上稍加扩展一些轻松风格的描述或元素，主调还是输入主题内容
- 你根据主要内容自行判断是否建议我新增和采用其它风格？

### 7. 强调格式

✅ 使用加粗强调关键词

```markdown
**能跑就行，不问出处。**

**证据 > 速度**
```

✅ 使用代码内联标记

```markdown
`CLAUDE.md`、`skills/`、`.env`
```
