在使用 Claude Code 处理复杂任务时，有一个关键指标直接影响着 AI 的输出质量和稳定性——**上下文占用率**。本文分享一套完整的上下文管理方法，帮你保持 AI 始终在最佳状态。

---

## 🎯 为什么上下文如此重要？

Claude Code 的上下文窗口虽然强大，但**不是无限的**。随着对话轮数和工具调用的累积，系统会自动压缩历史消息以适配上下文限制。

> **关键洞察**：上下文占用越高，模型越可能不稳定和出错

更关键的是——**不要做甩手掌柜**。Boris（Claude Code 开发创始人）的团队在复杂任务中，**每 30-45 分钟就做一次上下文重置**。

---

## 📊 上下文监测方法

### 使用 `/context` 命令

**随时查看当前占用率**：

```
/context
```

**输出示例**：
```
Current context usage: 45% of model limit
```

**建议阈值**：
| 占用率 | 状态 | 行动 |
|--------|------|------|
| <30% | ✅ 健康 | 继续 |
| 30-60% | ⚠️ 关注 | 考虑整理 |
| >60% | 🔴 警告 | 准备压缩 |
| >80% | 🚨 危险 | 立即处理 |

---

## 🛠️ 上下文处理方法

### 方法一：自动压缩（`/compact`）

**适用场景**：占用率较高，想快速整理

**执行方式**：
```
/compact
```

**特点**：
- ✅ 快速、自动化
- ✅ 智能化压缩历史消息
- ⚠️ 压缩逻辑不可控，可能丢失关键细节

> **经验**：自动压缩是"黑盒"，不如人工确认后操作精准

---

### 方法二：人工整理（推荐⭐）

**适用场景**：复杂任务、关键节点、需要精准控制

**执行步骤**：

#### 1️⃣ 中断当前逻辑

```
Ctrl + C
```

#### 2️⃣ 切换到 Plan 模式

```
/plan
```

#### 3️⃣ 输出总结文档

输入指令：
```
停止分析，总结我们目前做了什么，还剩哪些任务，当前有哪些问题
```

AI 会输出类似：
```markdown
## 已完成
- [x] 项目结构分析
- [x] 核心模块识别
- [x] 配置文件整理

## 待完成任务
- [ ] 重构 utils 模块
- [ ] 补充单元测试
- [ ] 更新文档

## 当前问题
- 配置文件中存在未解析的占位符
- 部分依赖版本需要升级
```

#### 4️⃣ 确认和修改总结文档

**仔细检查输出的内容**：
- 已完成事项是否准确？
- 待办列表是否完整？
- 问题描述是否清晰？

用 `Edit` 工具修改不完善的部分。

#### 5️⃣ 重启会话

**选项 A**：清空当前会话，基于修改后的 `.md` 文档重新开始

**选项 B**：手动复制文档内容，开启新会话继续

---

## 🔄 两种方法对比

| 维度 | 自动压缩 | 人工整理 |
|------|----------|----------|
| 速度 | ⚡ 快 | 🐢 较慢 |
| 精准度 | ⚠️ 不可控 | ✅ 完全可控 |
| 适用场景 | 简单任务/临时清理 | 复杂任务/关键节点 |
| 信息丢失风险 | 中 | 低 |

---

## 💡 最佳实践建议

### 1. 主动监测，不要等警告

> **不要等系统自动压缩** —— 那时可能已经丢失了关键上下文

养成习惯：
- 每小时执行一次 `/context`
- 发现占用率>50% 就考虑整理
- 复杂任务每完成一个阶段就整理

### 2. 复杂任务的节奏控制

**推荐节奏**：
```
开始任务 → 每 30-45 分钟 → 人工整理 → 继续推进
```

**为什么是这个频率？**
- 30-45 分钟是一个自然的注意力周期
- 此时上下文占用通常在 40-60%
- 整理成本最低，信息损失最小

### 3. 哪些情况必须人工整理？

| 场景 | 原因 |
|------|------|
| 架构级决策 | 自动压缩可能误解关键权衡 |
| 复杂 Bug 修复 | 调试过程中的试错信息很重要 |
| 多文件重构 | 文件间依赖关系容易被压缩丢失 |
| 核心逻辑修改 | 需要保持完整的决策链条 |

## 📝 上下文管理检查清单

开始复杂任务前：
- [ ] 了解 `/context` 命令
- [ ] 知道如何切换到 Plan 模式
- [ ] 准备好记录总结的 `.md` 文件

任务进行中：
- [ ] 每小时检查一次上下文占用
- [ ] 超过 50% 考虑整理
- [ ] 完成关键阶段后主动总结

需要整理时：
- [ ] 优先选择人工整理
- [ ] 仔细审核 AI 输出的总结
- [ ] 确保待办事项完整准确

## 🌅 尾声

上下文管理的本质不是"节省资源"，而是**保持 AI 始终在清晰、稳定的状态下工作**。

> **因为最好的系统，不是能塞进最多信息，而是能在最清晰的状态下做决策。**

主动掌控上下文，让 Claude Code 始终为你的复杂任务保持最佳状态。
