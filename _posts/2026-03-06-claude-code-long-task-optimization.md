处理复杂长任务时，即使是 Claude Code 这样的强大陆续也可能"跑偏"。本文分享一套简单有效的干预方法，帮你保持任务在正确轨道上。

---

## 🎯 核心问题

在 Plan 模式下，多轮整理需求文档后开始执行时，由于任务阶段太多、步骤太细、待办事项繁杂，**不能从头到尾完全不管**。

> **关键洞察**：长任务需要" checkpoints "，而不是" set and forget "

---

## 📋 四步干预法

### 1️⃣ 当前总结

**触发时机**：完成某个阶段或关键步骤后

**输入指令**：
```
当前总结：总结已经完成的部分，整理待办
```

**作用**：
- 强制 AI 停下来回顾进度
- 清晰化已完成 vs 待完成的边界
- 避免"无意识推进"导致的偏离

---

### 2️⃣ 阶段测试验证

**触发时机**：收到总结后，验证已完成工作

**输入指令**：
```
阶段测试验证：基于待办的整理，你先把当前已经完成的部分自我测试验证一下前面部分的工作成果
```

**作用**：
- 防止带着 bug 进入下一阶段
- 早期发现问题，降低返工成本
- 建立"完成即正确"的信心

---

### 3️⃣ 整装待发

**触发时机**：验证通过，准备进入下一阶段

**输入指令**：
```
整装待发，重新整理完成情况和待办：由于已经验证了前面完成工作大体通过了验证，根据目前完成情况重新整理 TODO.md 看是否合理，重点看接下来待办步骤和内容是否正确
```

**作用**：
- 动态调整待办列表，保持合理性
- 确保下一步方向正确
- 可能发现之前规划中的遗漏或错误

---

### 4️⃣ 介入微调

**触发时机**：发现总结与待办列表存在问题时

**输入指令**：
```
【指出发现的关键问题，描述要点即可】
```

**示例**：
```
待办第 3 步依赖的配置还没创建，应该先完成环境准备
测试用例只覆盖了主流程，边界情况需要补充
```

**作用**：
- 及时纠正偏差
- 避免问题累积到后期爆发
- 保持任务整体质量

---

## 🔄 循环使用

这四个步骤不是线性的，而是一个**循环干预系统**：

```
完成阶段 → 总结 → 验证 → 调整待办 → [微调] → 继续下一阶段 → 重复...
```

> **最佳实践**：每个主要阶段结束后，至少执行 步骤 1 + 步骤 2

---

## 💡 使用建议

| 场景 | 推荐操作 |
|------|----------|
| 小型任务（<5 文件） | 仅在完成时执行步骤 1 |
| 中型任务（5-15 文件） | 每 2-3 个阶段执行一次完整循环 |
| 大型任务（>15 文件） | 每个阶段结束后执行完整循环 |
| 高风险任务（核心逻辑） | 每个子步骤后都执行步骤 1+2 |

---

## 🌅 尾声

长任务管理的核心不是"管得更细"，而是**在关键节点停下来确认方向**。

> **因为最远的弯路，往往始于"应该没问题"的假设。**

这套方法的价值在于：用最少的干预，换取最大的确定性。
