**Nobtodo** 是我最近开源的一个轻量本地待办工具。

它的定位很简单：本机单人使用，数据保存在项目目录里，不依赖账号体系，也不需要云端同步。打开本地网页，就可以在一个画布工作台里管理待办、列表、图片附件和提醒。

> **轻量、本地、够用，是 Nobtodo 的核心取向。**

开源地址：<https://gitee.com/nobt/nobt-todo>

---

## 🧩 产品定位

Nobtodo 默认运行在 `http://127.0.0.1:8189`，数据存储在项目的 `data` 目录中，包括任务数据、图片附件、缩略图和备份文件。

它更适合这些场景：

| 场景 | Nobtodo 的处理方式 |
|------|--------------------|
| 本机管理个人任务 | 数据直接落在本地项目目录 |
| 多个事项并行推进 | 用多个卡片列表分组管理 |
| 需要灵活摆放任务 | 使用可缩放、可拖拽的网格画布 |
| 待办里有截图或补充说明 | 支持说明、列表语法和图片附件 |
| 需要简单提醒 | 到期后在右下角提醒面板显示 |

它不是大型团队协作系统，也不是云端项目管理平台，而是一个偏个人工作台的本地待办工具。

## 🖼️ 界面预览

### 画布工作台与提醒面板

![Nobtodo 画布工作台与提醒面板](/img/blog/nobtodo-open-source/1.png)

中间区域是可缩放、可拖拽平移的网格画布。多个待办卡片可以自由摆放，支持不规则尺寸和位置布局。

右下角提醒面板常驻展示到期提醒，支持单条确认、关闭和全部确认。

### 左侧列表导航

![Nobtodo 左侧列表导航](/img/blog/nobtodo-open-source/2.png)

左侧侧边栏展示全部卡片列表、未完成数量、颜色标识和删除入口。点击列表可以定位到对应画布卡片，侧边栏也可以收起，让工作区更宽。

### 搜索与拼音检索

![Nobtodo 搜索与拼音检索](/img/blog/nobtodo-open-source/3.png)

顶部悬浮搜索框支持任务和列表搜索，同时支持中文、拼音全拼和拼音首字母检索。日常使用时，也可以通过 `Ctrl+F` 快速唤起搜索。

### 暗黑主题

![Nobtodo 暗黑主题](/img/blog/nobtodo-open-source/4.png)

Nobtodo 提供明亮和暗黑主题，覆盖画布、卡片、侧边栏、提醒面板、滚动条和工具栏等主要界面元素。

### 设置面板

![Nobtodo 设置面板](/img/blog/nobtodo-open-source/5.png)

设置面板支持主题切换、新增任务顺序设置、数据目录查看、备份导出和备份导入。

## ✅ 功能概览

Nobtodo 当前已经覆盖一个本地待办工具的主要使用闭环：

- 单窗口工作台，左侧列表导航可展开或收起。
- 中间区域为可缩放、可拖拽平移的网格画布。
- 卡片支持自由拖拽、调整大小、折叠/展开、颜色修改和删除。
- 支持 `Ctrl` + 鼠标单击多选多个卡片，并作为一组同步移动。
- 卡片可一键自动整理，执行前会二次确认。
- 卡片可不规则摆放，也可以通过辅助线进行中线和底部对齐。
- 画布支持最大化展示，最大化后隐藏顶部栏和抽屉，仅保留画布区域。
- 设置中可控制画布是否显示背景网格。
- 待办可在卡片内直接新增、编辑、完成、删除和调整顺序。
- 新增待办默认顺序追加，也可在设置中改为倒序添加。
- 删除图片、删除卡片、删除待办支持 `Ctrl+Z` 撤销。
- 待办支持补充说明，补充说明支持 `- `、`1. `、`a. ` 这类列表语法。
- 待办支持图片附件，图片显示为紧凑缩略图，双击可按顺序预览。
- 待办支持提醒时间，到期后在右下角提醒面板显示。
- 支持搜索任务和列表，支持中文、拼音全拼、拼音首字母。
- 支持导出和导入单个 zip 备份包，旧版 JSON 备份仍可导入。

## ⌨️ 常用快捷键

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+F` | 打开顶部搜索框 |
| `Alt+N` | 新增；存在激活卡片时新增待办，没有激活卡片时新增卡片列表 |
| `Alt+S` | 展开或收起侧边栏，最大化画布时也可使用 |
| `Alt+=` / `Alt++` | 以当前激活卡片为中心放大画布 |
| `Alt+-` | 以当前激活卡片为中心缩小画布 |
| `Alt+0` | 恢复 100% 缩放，并把画布滚动到左上角 |
| `Enter` | 在激活待办上打开并聚焦补充说明 |
| `Ctrl+Z` | 撤销最近一次删除图片、删除待办或删除卡片 |

## 🚀 如何使用

先获取项目代码：

```bash
git clone https://gitee.com/nobt/nobt-todo.git
cd nobt-todo
```

安装依赖并启动：

```bash
npm install
npm run build
npm run web
```

然后打开：

```text
http://127.0.0.1:8189
```

数据目录默认在：

```text
D:\AiCoding_project\nobt-todo\data\
```

如果你把项目放在其它位置，数据也会跟随当前项目目录保存到对应的 `data` 目录里。

---

## 🛠️ 开发方式

如果想参与开发，可以同时启动本地服务和前端开发页：

```bash
npm run web
npm run dev
```

开发页面地址：

```text
http://127.0.0.1:1420
```

API 由 `8189` 端口的本地服务写入 `data` 目录。
